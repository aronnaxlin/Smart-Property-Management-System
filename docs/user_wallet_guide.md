Card和Fee要做合理的关联，现在产生了Fee却无法使用Card充值，那么这个缴费逻辑就只成为了文本输出。为了模仿更真实的逻辑，我们这里设立两种卡模式。

1. 水卡、电卡单独管理，用于支付水费、电费 

2. 用户钱包用于支付剩余的费用，如物业费、 暖气费等，直接从用户钱包扣除金额，且可以充值水卡、电卡余额。

规则如下：

1. 水卡、电卡的余额可以通过用户钱包充值；而水费、电费不能通过用户钱包直接扣除，只能从水卡、电卡扣除。 
2. 当使用用户钱包支付的账单未缴费时，禁止充值水卡、电卡余额。如用户充值水卡时，有物业费/暖气费未缴，提示用户先缴纳对应账单，再开放充值水卡。
3. 一个用户只有一个用户钱包；若用户拥有多套房产，则只允许一个用户钱包，该用户钱包允许**分别**支付不同房产对应的物业费/暖气费等，允许**分别**充值对应的水卡、电卡等。
4. 一套房产只有一张水卡、一张电卡。
5. 一套房产可产生多笔物业费/暖气费等。
6. 用户钱包的充值方法暂为模拟输入，即用户输入合法数值即可充值。
